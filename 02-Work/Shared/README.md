# Shared Documentation

## Purpose
Repository untuk dokumentasi yang digunakan bersama antara team MRG dan UPG.

## Contents

### Standards
Coding standards, API guidelines, dan database conventions yang harus diikuti kedua team.

### Patterns
Design patterns, architectural patterns, dan best practices untuk microservices development.

### Integration
Dokumentasi integrasi antara MRG dan UPG, event flows, dan API gateway patterns.

### Glossary
Business terms dan technical terms yang digunakan di kedua platforms.

## Usage Guidelines
1. **Standards** harus diikuti oleh semua services
2. **Patterns** adalah recommended practices, bisa di-adapt sesuai kebutuhan
3. **Integration docs** wajib diupdate ketika ada perubahan contract antar team
4. **Glossary** harus di-maintain untuk menghindari ambiguity

## Contribution
Setiap perubahan di shared documentation harus melalui review dari architect dan mendapat approval dari kedua team leads.
